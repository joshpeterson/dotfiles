# OpenCode Agent Instructions

## Git Commits

**CRITICAL**: Do not commit changes on your own without my explicit instructions.

### Commit Behavior Rules

1. **Always ask before committing**: I want to review code changes first. Do not commit unless I explicitly tell you to.

2. **Single commit authorization**: If I ask you to commit one specific change, this does **NOT** grant permission to commit other changes. Each change requires separate authorization.

3. **Ask about each change**: You should ask me about each change you want to commit, unless I explicitly tell you to "commit all changes".

4. **Explicit consent required**: Never assume I want to commit anything. Always wait for my confirmation before running any git commit commands.

### Examples of What NOT to Do

- ❌ Making a change and immediately committing it without asking
- ❌ Committing multiple changes after I asked you to commit just one
- ❌ Assuming "commit the changes" means "commit everything you did"

### Examples of What TO Do

- ✅ "I've made changes to X and Y. Should I commit them?"
- ✅ "I fixed the bug. Would you like me to commit this change?"
- ✅ "I have 3 changes ready. Which ones would you like me to commit?"
